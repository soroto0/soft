# YouTube Video Pipeline

Сценарий → озвучка (Edge TTS / Amazon Polly) → стоки (Pexels/Pixabay) → субтитры (Whisper) → папка для Premiere Pro.

Структура:
- `core.py` — вся логика пайплайна (общая для CLI и GUI)
- `pipeline.py` — CLI: `python pipeline.py`
- `app.py` — GUI «Контент-фабрика»: `python app.py`

## Установка (один раз)

1. Установи Python 3.10+ и FFmpeg:
   - Windows: `winget install ffmpeg` (или скачай с ffmpeg.org и добавь в PATH)
   - Проверка: `ffmpeg -version`

2. Установи зависимости:
   ```
   pip install -r requirements.txt
   ```

3. Скопируй `.env.example` в `.env` и впиши ключи:
   - AWS (Polly): aws.amazon.com → IAM → пользователь с политикой AmazonPollyFullAccess
   - Pexels: pexels.com/api
   - Pixabay: pixabay.com/api/docs

## Оверлеи: движок Remotion (опционально, кинокачество)

Моушн-графику (плашки, счётчики, выноски, popup, бары, таймлайн) может
рендерить Remotion — React-анимации через браузерный движок, качество
уровня After Effects. Нужен Node.js 18+:

```
cd remotion && npm i
```

Дальше всё автоматически: если Node и `remotion/node_modules` на месте,
оверлеи рендерит Remotion; иначе — встроенный Pillow (быстрее, проще).
Принудительный выбор — в settings.json: `"overlay_engine": "auto" |
"remotion" | "pillow"`. Упавший оверлей всегда откатывается на Pillow,
рендер не падает.

## Использование (каждое видео)

1. Положи английский текст сценария в `script.txt`
2. Заполни `scenes.txt` — одна сцена на строку:
   ```
   dark city street at night, rain | type: video | count: 2
   old newspaper archive | type: image
   police car lights | type: video
   ```
   `count: N` — скачать N разных клипов на сцену (до 5). Клипы выбираются
   случайно из топ-15 результатов; уже использованные в прошлых видео
   не повторяются (история — `used_media.json`).
   `type: gen` — сгенерировать картинку через ИИ: сначала Agnes
   (AGNES_API_KEY), при ошибке — Gemini (GEMINI_API_KEY; у бесплатного
   тарифа Gemini квота на картинки = 0, нужен биллинг).

## Авто-раскадровка (материал по таймлайну озвучки)

Вместо ручного списка сцен материал подбирается под то, о чём говорится
в каждый момент: субтитры дают таймкоды, озвучка режется на планы по ~6 с,
из текста плана извлекаются ключевые слова, под план качается видео нужной
длины (или фото + Ken Burns ровно на длину плана; если стока нет и задан
ключ Gemini — картинка генерируется).

- GUI: вкладка «Видеоматериал» → «Собрать по субтитрам»
- CLI: `python pipeline.py --storyboard` (`--beat 8` — длина плана)

Результат: `storyboard/` с клипами, `timeline.json` и `sequence.xml` —
в Premiere Pro: **File → Import → sequence.xml** — готовый таймлайн:
клипы стоят по таймкодам, озвучка (с музыкой, если подмешана) под ними.
Дальше — обычный монтаж: заменить неудачные планы, добавить переходы.
Если в видео есть реалистичные сгенерированные кадры, не забудь отметить
это при загрузке на YouTube (Altered content).
3. Запусти:
   ```
   python pipeline.py
   ```
4. Забери папку `output/` и импортируй в Premiere Pro:
   - `audio/voiceover.mp3` — озвучка
   - `video/`, `images/` — визуал (названия файлов = номер сцены)
   - `subs/voiceover.srt` — субтитры (File → Import)
   - `manifest.json` — список сцен и файлов

## Полезные флаги

```
python pipeline.py --tts edge     # озвучка бесплатным Edge TTS (без ключей)
python pipeline.py --rate -5      # темп речи на 5% медленнее
python pipeline.py --no-pauses    # без пауз между абзацами (Polly SSML)
python pipeline.py --music PATH   # подмешать музыку (папка или файл)
python pipeline.py --no-kenburns  # не делать клипы из картинок
python pipeline.py --skip-tts     # без озвучки (уже есть mp3)
python pipeline.py --skip-media   # без скачивания стоков
python pipeline.py --skip-subs    # без субтитров
```

## Агенты (Gemini + Agnes)

Роли: **тексты** (сценарий, сцены, умные запросы, SEO) — Gemini
(GEMINI_API_KEY), при ошибке — Agnes; **картинки** (type: gen, раскадровка) —
Agnes (AGNES_API_KEY), при ошибке — Gemini. Достаточно одного ключа —
второй просто добавляет запасной вариант.

- **Сцены через ИИ** — кнопка «Сцены через ИИ» (вкладки «Сценарий» и
  «Видеоматериал»): LLM читает каждый абзац сценария и составляет scenes.txt —
  что должно быть на экране (запрос для стока) и тип (video/image).
  Без ключа или при ошибке — локальная разметка по абзацам.
- **Черновик сценария** — вкладка «Сценарий»: тема + хронометраж (15–90 мин) →
  план из глав → главы пишутся по очереди, каждая продолжает предыдущую.
  ~150 слов/мин, стиль «без воды». Это черновик: вычитай и переработай
  перед озвучкой, сырой текст нейросети — «inauthentic content».
- **Умные запросы в раскадровке** — если ключ задан, поисковые запросы для
  стоков составляет LLM по смыслу фразы, а не по частоте слов
  (при ошибке — автоматический откат на ключевые слова).
- **SEO** — вкладка «Сборка»: 5 названий, описание и теги по сценарию
  (сохраняется в `seo.txt`).

## Авторендер (черновой mp4 без Premiere)

Вкладка «Авторендер» собирает output_final.mp4 из готовых материалов:
озвучка + субтитры (таймкоды фраз) + video/ + images/ + storyboard/.

- Смены кадров — по границам фраз; вопросы и цифры начинают новую сцену.
- Ритм-логика: вариация длительностей, первые 15 с — динамичный монтаж,
  каждые 60-120 с серия коротких врезок; seed фиксирован на проект
  (два проекта никогда не дают одинаковый рисунок монтажа).
- Движение в кадре: Ken Burns (4 направления), drift, push-in, микро-тряска.
- Переходы: пул из 13 (crossfade, fade black/white, swipe, zoom, wipe,
  slide, circle, dissolve, hard cut), взвешенный выбор без повторов подряд.
- Стилевые слои: зерно, виньетка, letterbox 2.35:1, VHS-пресет.
- Субтитры вшиваются (белые с чёрной обводкой, нижняя треть).
- Музыка: voiceover_music.mp3 подхватывается автоматически; выбор трека
  или подбор по настроению (подпапки dark/tense/neutral/uplifting в папке
  музыки) — с fade in/out и автодакингом.
- Настройки: 1080p/4K, 24/30/60 fps, интенсивность монтажа.

Рендер по сегментам с прогресс-баром; лог каждого шага в журнале.
CLI-запуска нет — это GUI-функция. Итог — черновик: доводи в Premiere.

## Против «inauthentic content»

- **Вариативность стоков**: клип берётся случайно из топ-15 результатов;
  `used_media.json` хранит историю — один клип никогда не попадёт в два видео.
- **`count: N`** — несколько разных клипов на сцену для плотного монтажа.
- **Ken Burns**: каждая картинка превращается в клип 1080p с движением камеры
  (наезд/отъезд/панорама, случайно) — кладётся в `video/*_kb.mp4`.
- **Живая озвучка**: темп речи (`--rate` / выбор в GUI), паузы между абзацами
  через SSML (Polly), движок `generative` звучит человечнее `neural`.
- **Музыка с автопригушением**: трек подкладывается под голос и приглушается,
  когда голос звучит (`voiceover_music.mp3`; чистый голос остаётся).
- **Чеклист** на вкладке «Сборка»: секунд на клип, музыка, история стоков.

Это убирает типичные флаги конвейерного контента, но не заменяет главного:
оригинальный сценарий и собственный монтаж.

## Удобства GUI

- Сценарий можно вставить прямо в поле (Ctrl+V или правой кнопкой мыши) —
  горячие клавиши работают и в русской раскладке.
- script.txt, scenes.txt и субтитры подхватываются из папки проекта
  автоматически (при запуске и смене папки).
- «▶ Прослушать голос» на вкладке «Озвучка» — короткий пример звучания
  до озвучки всего сценария.
- Журнал внизу окна пишет ход каждой операции с отметками времени.

## Примечания

- Первый запуск Whisper скачает модель (~150 МБ для base.en) — это нормально.
- Голоса Polly можно менять в .env: Matthew, Joanna, Stephen (en-US neural).
- Если сцена «НЕ НАЙДЕНО» — переформулируй ключевые слова проще (2-4 слова).
